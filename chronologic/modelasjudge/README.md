# ChronoLogic Free Generation Judging Strategy

Allowing the model to freely generate an answer is in a sense the fairest judging strategy, since the model receives no help

The model-as-judge workflow requires three separate judgments for each free-generated answer. First, an LLM judge (Claude Sonnet 4.6 for all non-Claude models; a GPT for Claude family models) evaluates two dimensions in a single forced-choice prompt: *question fit* — whether the answer is substantively correct — and *context fit* — whether the answer is consistent with the historical source (author, date, genre) specified in the metadata frame. It does this by comparing the model-generated answer to ground truth and deciding which is a better fit. If the model wins *or ties* the ground truth, it passes.

Second, a fine-tuned DeBERTa classifier trained to distinguish authentic period prose from LLM-generated infill evaluates *stylistic authenticity*, producing a continuous probability that the answer is period-authentic. A model gets a question "right" if its answer passes all three tests: question accuracy, context fit, and stylistic authenticity.

A missing style score (for short or factual answers where style is uninformative) counts as an automatic pass on that dimension.

Each judge is assigned a per-question reliability score before it scores any candidate model. For the LLM judge, reliability is computed empirically for each question by running the judge on all known ground-truth/distractor pairs in the benchmark (with both answer orderings tested), measuring what fraction of comparisons it gets right under the appropriate penalty rules — which vary by distractor type, since a same-book distractor is not actually "wrong" for context fit, only for question accuracy.

For the discriminative style judge, reliability is estimated at the group level, stratified by reasoning type and answer-length decile, because per-question sample sizes are too small for stable estimates. Right now, questions where LLM judge reliability falls at or below 0.65 are escalated to a human annotator; style questions below 0.70 receive NA and are excluded. [I would suggest setting both floors at .70.]

Human reliability is currently supplied as a single fixed value entered by the annotator at the start of a scoring session — this is an acknowledged limitation, since a properly calibrated human reliability estimate would require inter-rater comparison on a shared subsample, which we haven't done yet.

Final scores and uncertainty intervals are computed with a hierarchical Bayesian model implemented in PyMC. The key idea is that each judge judgment is treated not as ground truth but as a noisy signal: if a judge is right 92% of the time on a given question, an observed "pass" verdict is consistent with the model truly passing — but it also allows some probability that the judge got it wrong. The model works backward from all the observed verdicts (across all three scoring dimensions) to estimate the most probable true pass rates, accounting for the fact that question fit and context fit tend to rise and fall together while style varies independently. Crucially, this procedure also corrects for a systematic downward bias: when three imperfect judges must all agree for a question to "pass," even small error rates multiply together and compress scores toward zero, understating how well the model actually did. The resulting posterior distributions give us both a bias-corrected point estimate and a credible interval reflecting residual uncertainty. In practice those intervals have been narrow: for GPT-5.4 on benchmark v0.2 (N ≈ 210 questions) the 95% interval on overall binary accuracy was 0.436–0.481 (width ~0.045); for a weakly-performing fine-tuned model (talkie-1930) it was 0.029–0.047 (width ~0.018). Style accuracy intervals are wider — roughly ±0.04–0.05 — reflecting the style judge's lower and more variable reliability (~0.75 average vs. ~0.95 for the LLM judge).
